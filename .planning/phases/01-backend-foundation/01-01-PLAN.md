<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API | Untrusted request bodies, query params, and path params enter FastAPI route handlers. |
| Vapi -> webhook | External webhook payloads may be spoofed, replayed, or duplicated. |
| API -> database | Concurrent sessions and persistence writes must not corrupt household, dependent, or event state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T | household/dependent/timeline routes | mitigate | Validate all input with Pydantic/SQLModel schemas, return explicit 4xx errors at the route boundary, and cover negative cases in endpoint tests. |
| T-01-02 | S/T | voice webhook | mitigate | Keep HMAC signature verification, rate limiting, and idempotency guards active; reject invalid signatures and duplicate tool calls in tests before they reach business logic. |
| T-01-03 | D | async DB session handling | mitigate | Use commit/rollback/close semantics in `get_session`, exercise real DB integration tests, and keep schedule generation idempotent to avoid duplicate event rows. |
</threat_model>
