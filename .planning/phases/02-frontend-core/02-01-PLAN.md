Phase 2 — Frontend Core Plan
=============================

Goal
----
Deliver a production-ready frontend core: auth, household/dependent flows, timeline view, scanner flow, voice entry, and CI for deployment to Vercel.

Tasks
-----
1. Auth: integrate login flow
   - Implement login page calling `/api/v1/login` and store JWT in cookie/localStorage.
   - Protect pages with auth guard and `getServerSideProps` where needed.
   - Verify token refresh behavior and session expiration.

2. Household & Dependents CRUD
   - Build household overview, dependent creation form, and edit flows.
   - Wire API client to backend endpoints; add optimistic UI where appropriate.

3. Timeline & Activity Feed
   - Implement timeline view using existing components; add filters and pagination.
   - Ensure accessibility and responsive layouts.

4. Scanner + OCR Flow
   - Integrate scanner UI to upload images to backend OCR endpoint (or client-side fallback).
   - Add progress & retry UX.

5. Voice Button + Vapi Integration (dev mode)
   - Add Voice FAB to trigger demo voice flows pointing to local backend `/api/v1/voice/webhook`.
   - Ensure webhook secret is configurable in dev and not committed.

6. Tests & Quality
   - Add unit tests for key components (Vitest + React Testing Library).
   - Add an E2E smoke test (Playwright) validating login → create household → view timeline.

7. CI & Deployment
   - Add Vercel project config and environment variable mapping (`NEXT_PUBLIC_API_URL`).
   - Add a GitHub Action to run lint/tests and deploy preview branches to Vercel.

Verification
------------
- All unit tests pass and E2E smoke test succeeds.
- Auth flow works end-to-end against local backend and deployed backend.
- Key user flows (create household, add dependent, view timeline, scanner upload) succeed.

Threats & Mitigations
---------------------
- Secrets in repo: ensure `.env` and `.env.example` do not contain real secrets; use Vercel env vars.
- Token leakage: store tokens in HttpOnly cookies where possible.
- Accessibility regressions: run axe checks on key pages.

Next Steps
----------
- Start Wave A: Auth + Household CRUD (2–3 days)
- After Wave A completes, run frontend E2E smoke and proceed with Wave B (Timeline + Scanner + Voice)
