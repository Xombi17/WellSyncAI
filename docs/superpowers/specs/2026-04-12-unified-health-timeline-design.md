# Unified Health Timeline Expansion — Design Spec

**Date:** 2026-04-12
**Status:** Approved
**Approach:** Extend existing HealthEvent + schedule engine to cover pregnancy care, medicine adherence, and child growth tracking

---

## Problem

WellSync Vaxi currently covers child vaccination schedules only. Families need the same proactive, voice-first health memory for pregnancy checkups, daily medicine adherence, and child growth monitoring. Building three separate feature areas would duplicate infrastructure and fragment the user experience.

## Solution

Extend the existing unified health timeline to include three new health domains. Every new event type — prenatal visit, medicine dose, growth check — becomes a `HealthEvent` row on the same timeline families already use. The same voice assistant, status computation, and reminder logic work across all domains without duplication.

---

## Data Model

### New EventCategory values

Add to the existing `EventCategory` enum:

| Value | Purpose |
|-------|---------|
| `prenatal_checkup` | Pregnancy-related visits and tests |
| `medicine_dose` | Recurring medicine adherence events |
| `growth_check` | Height/weight/milestone measurement prompts |

### New model: PregnancyProfile

```python
class PregnancyProfile(SQLModel, table=True):
    id: str                      # UUID
    household_id: str             # FK -> Household
    lmp_date: date                # Last menstrual period
    expected_due_date: date       # Computed: LMP + 280 days
    high_risk_flags: str = ""     # Comma-separated flags
    completed: bool = False       # True after delivery
    created_at: datetime
    updated_at: datetime
```

Computed properties (not stored):
- `pregnancy_week`: days since LMP / 7
- `trimester`: 1st (weeks 1-12), 2nd (13-26), 3rd (27-40+)
- `days_until_due`: expected_due_date - today

### New model: MedicineRegimen

```python
class MedicineRegimen(SQLModel, table=True):
    id: str                      # UUID
    dependent_id: str            # FK -> Dependent
    medicine_name: str
    dosage: str                  # e.g., "500mg", "1 tablet"
    frequency: FrequencyType     # daily, twice_daily, three_daily, weekly, as_needed
    start_date: date
    end_date: date | None = None # None = ongoing
    safety_bucket: str = ""      # From existing medicine_safety classifier
    prescribing_note: str = ""
    active: bool = True
    created_at: datetime
    updated_at: datetime

class FrequencyType(str, Enum):
    DAILY = "daily"
    TWICE_DAILY = "twice_daily"
    THREE_DAILY = "three_daily"
    WEEKLY = "weekly"
    AS_NEEDED = "as_needed"
```

### New model: GrowthRecord

```python
class GrowthRecord(SQLModel, table=True):
    id: str                      # UUID
    dependent_id: str            # FK -> Dependent
    recorded_date: date
    height_cm: float | None = None
    weight_kg: float | None = None
    head_circumference_cm: float | None = None  # 0-2 years only
    milestone_achieved: str | None = None
    notes: str = ""
    created_at: datetime
```

GrowthRecord is NOT a HealthEvent. It stores measurement data. The timeline shows `growth_check` events (e.g., "Monthly weight check due") which prompt the user to create a GrowthRecord.

### No changes to existing models

- `Household` and `Dependent` are unchanged
- `DependentType` already includes `pregnant`
- `HealthEvent` gains new categories but no new columns
- `schedule_key` pattern continues for idempotency

## Schedule Engine

### Pregnancy schedule (data/prenatal_schedule.json)

WHO ANC visit schedule based on LMP date:

| Visit | Week | Tests |
|-------|------|-------|
| Booking visit | 12 | Blood type, hemoglobin, HIV, syphilis, urine |
| 2nd visit | 20 | Ultrasound (anomaly scan) |
| 3rd visit | 26 | Blood pressure, urine, fundal height |
| 4th visit | 30 | Blood pressure, urine, fetal position |
| 5th visit | 34 | Blood pressure, urine, growth scan |
| 6th visit | 36 | Blood pressure, urine, vaginal exam |
| 7th visit | 38 | Blood pressure, urine, fetal monitoring |
| 8th visit | 40 | Full assessment, delivery planning |

Generated by `generate_prenatal_schedule(lmp_date)` — same idempotent pattern as `generate_child_schedule`.

### Medicine dose events (computed from regimen)

Unlike vaccinations, medicine doses are user-defined. When a `MedicineRegimen` is created:

1. Engine generates `HealthEvent` rows with category `medicine_dose` based on `frequency` + `start_date`
2. Only generates events for the next 30 days (rolling window)
3. On timeline fetch, if the window is expiring, regenerates the next 30 days
4. `schedule_key` format: `med-{regimen_id}-{date}-{dose_number}`
5. For `as_needed` frequency, no events are generated — the regimen appears as a reference card only

### Growth check schedule (data/growth_check_schedule.json)

Age-based check intervals from WHO standards:

| Age range | Check interval |
|-----------|----------------|
| 0-6 months | Monthly |
| 6-12 months | Every 2 months |
| 1-3 years | Quarterly |
| 3-5 years | Every 6 months |
| 5+ years | Annually |

Each growth check prompts: weight, height, head circumference (0-2 years), and age-appropriate developmental milestone.

Generated by `generate_growth_check_schedule(dob)` — same idempotent pattern.

### Engine changes

**rules.py additions:**
- `generate_prenatal_schedule(lmp_date) -> list[ScheduleEntry]`
- `generate_growth_check_schedule(dob) -> list[ScheduleEntry]`
- `generate_medicine_events(regimen: MedicineRegimen) -> list[ScheduleEntry]`

**engine.py additions:**
- `save_prenatal_schedule(pregnancy: PregnancyProfile, session) -> list[HealthEvent]`
- `save_medicine_events(regimen: MedicineRegimen, session) -> list[HealthEvent]`
- `save_growth_check_schedule(dependent: Dependent, session) -> list[HealthEvent]`

All follow the same idempotent `schedule_key` pattern as existing `generate_and_save_schedule`.

## API Endpoints

### Pregnancy

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/pregnancy` | Create pregnancy profile from LMP date |
| GET | `/api/v1/pregnancy/{household_id}` | Get active pregnancy + computed week/trimester |
| PATCH | `/api/v1/pregnancy/{household_id}` | Update pregnancy (mark completed, add risk flags) |

### Medicine Regimens

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/medicines` | Add medicine regimen for a dependent |
| GET | `/api/v1/medicines/{dependent_id}` | List active medicine regimens |
| PATCH | `/api/v1/medicines/{regimen_id}` | Update regimen (dosage, frequency, end date) |
| DELETE | `/api/v1/medicines/{regimen_id}` | Deactivate regimen |

### Growth Records

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/growth/{dependent_id}` | Record a growth measurement |
| GET | `/api/v1/growth/{dependent_id}` | Get growth history |

### Timeline changes

- `GET /api/v1/timeline/{dependent_id}` gains optional query param `?category=` (vaccination, prenatal_checkup, medicine_dose, growth_check, or all)
- Response shape unchanged — new event types are `HealthEvent` rows with different categories

## Voice Extensions

### New tool calls for Vapi agent

| Tool | Purpose | Parameters |
|------|---------|------------|
| `record_medicine_taken` | Mark a medicine dose event as completed | `regimen_id`, `date` |
| `record_growth` | Voice-guided growth measurement recording | `dependent_id`, `weight_kg?`, `height_cm?` |
| `check_pregnancy_status` | Current week, next ANC visit, trimester summary | `household_id` |

### Voice context assembly changes

`voice.py` context payload now includes:
- Pregnancy profile for the household (if active)
- Active medicine regimens for the dependent
- Recent growth records

`answer_voice_question` already works for any HealthEvent — prenatal and growth check events get explanations without code changes.

### AI explanation additions

| Function | Purpose |
|----------|---------|
| `explain_pregnancy_visit` | Trimester-specific visit guidance |
| `interpret_growth` | Compare measurement against WHO percentiles, return simple language |

Both follow the existing fallback pattern: static text on AI failure.

## Frontend

### New pages

| Route | Purpose |
|-------|---------|
| `/pregnancy` | Active pregnancy dashboard — week, trimester bar, next visit card, ANC timeline |
| `/pregnancy/setup` | Enter LMP date, confirm due date, start tracking |
| `/medicines` | Medicine regimen list for a dependent, adherence streaks |
| `/medicines/add` | Add new medicine (name, dosage, frequency, dates) |
| `/growth/{dependent_id}` | Growth chart + latest measurements + milestone checklist |

### Timeline page changes

- Category filter tabs at top: All | Vaccinations | Pregnancy | Medicines | Growth
- Each category has distinct color and icon in the timeline feed
- "Next due" card reflects whichever event is soonest across all categories

### New components

| Component | Purpose |
|-----------|---------|
| `PregnancyCard` | Current week, trimester, days until next visit, baby size comparison |
| `MedicineCard` | Medicine name, dosage, next dose time, adherence streak |
| `GrowthChart` | Line chart (weight + height over time) with WHO percentile bands |
| `MilestoneChecklist` | Age-appropriate developmental milestones with checkboxes |
| `CategoryFilter` | Timeline category tabs (reused across views) |

### Voice UX

No new voice UI. Existing `VoiceFAB` handles all new domains naturally. Voice commands extend: "When is my next checkup?", "Did I take my medicine today?", "How is my baby growing?"

### Mobile-first

All new pages follow existing responsive patterns (320px+). Growth chart uses simple SVG — no heavy charting library. Pregnancy dashboard uses card-based layout matching existing timeline.

## Error Handling

All new endpoints follow existing pattern: explicit 4xx at route boundary, safe fallback for AI paths.

| Endpoint | Validation |
|----------|------------|
| Pregnancy create | Reject LMP dates in the future or beyond 42 weeks |
| Medicine regimen create | Require dosage + frequency, reject end_date before start_date |
| Growth record create | Reject implausible values (weight > 200kg, height > 250cm, head > 60cm) |
| All | Return explicit 4xx with descriptive messages |

## Safety Rules

- Pregnancy schedule: strictly from WHO ANC guidelines JSON — never LLM-generated
- Medicine adherence: reminders from regimen definition only — no LLM suggesting medicines
- Growth percentiles: computed from WHO standard tables — not AI-interpolated
- All medical content: "consult your doctor" when uncertain
- Medicine regimens: link to existing safety classifier for interaction warnings
- No diagnosis, no treatment recommendations, no autonomous medical decisions

## Scope Boundaries

### In scope (this milestone)

- Pregnancy profile CRUD + ANC schedule generation
- Medicine regimen CRUD + dose event generation (30-day rolling window)
- Growth record CRUD + growth check schedule generation
- All three domains on the unified timeline with category filtering
- Voice tool calls for all three domains
- Frontend pages for pregnancy dashboard, medicine list, growth tracking

### Out of scope (future milestones)

- Handwritten prescription parsing
- Allergy cross-checking against medicine regimens
- Pregnancy week-based medicine warnings (beyond existing safety classifier)
- Medicine reminder integration into household health timeline (separate from adherence)
- ASHA worker mode / community health worker dashboards
- WhatsApp/IVR notification channels
- Multi-tenant or clinic-level accounts
- Import/export integrations with external health platforms
